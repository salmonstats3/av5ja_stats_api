## API

ここでは実装済みAPIの詳細を解説します。

### Schedules

全てのエンドポイントで認証不要です。

|     エンドポイント     | メソッド | バージョン | 認証 |
| :--------------------: | :------: | :--------: | :--: |
|       schedules        |   GET    |     v1     | 不要 |
|       schedules        |   GET    |     v2     | 不要 |
|       schedules        |   POST   |     v1     | 不要 |
| schedules/:schedule_id |   GET    |     v1     | 不要 |

スケジュールIDがユニークかつ計算可能な値になったので4.0.0未満のAPIとは互換性がありません。V2となっているものはS3+で利用しているため互換性のために残していますが、将来的に削除予定です。

スケジュールはイカリング3ビューワから誰かが閲覧すれば自動で追加されるので、特に意識してこのAPIを叩く必要はありません。

### Results

リザルトはアップロードは認証不要ですが、取得には必要になります。認証のシステムは現段階ではまだ完成していないので、自身のリザルトを取得する方法はないことになります。

|   エンドポイント   | メソッド | バージョン | 認証 |
| :----------------: | :------: | :--------: | :--: |
|      results       |   POST   |     v1     | 不要 |
|      results       |   POST   |     v2     | 不要 |
|      results       |   GET    |     v1     | 必要 |
| results/:result_id |   GET    |     v1     | 必要 |

### Histories

4.0.0で新たに実装されたエンドポイントです。

| エンドポイント | メソッド | バージョン | 認証 |
| :------------: | :------: | :--------: | :--: |
|   histories    |   POST   |     v1     | 不要 |

`CoopHistoryQuery`で取得したデータをそのまま渡せばプライベートバイトのスケジュールが追加されます。

### Users

4.0.0で新たに実装されたエンドポイントです。

| エンドポイント | メソッド | バージョン | 認証 |
| :------------: | :------: | :--------: | :--: |
|     users      |   POST   |     v1     | 必要 |
|     users      |   GET    |     v1     | 必要 |
|     users      |  PATCH   |     v1     | 必要 |
| users/user_id  |   GET    |     v1     | 必要 |

まだ内容が決まっていないので特に気にしなくて大丈夫です。全てのエンドポイントで認証が必要です。
